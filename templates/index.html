Code Highlights

I am using the Flask framework to deploy the web application, GitHub as my source control provider, and PythonAnywhere to host and display my site all over the world.
From Flask, I'm importing the following:

  1. Flask to instantiate the application
  2. render_template: used for routing as well as to indicate which HTML files are served at each endpoint
  3. request: to handle the form request in the Contact page.

I am also importing the built-in modules smtplib and email.message to send an email directly to me when the form is sent.

After the routing to serve the homepage is done, I focused on the Contact page.
I chose to use the POST request instead of GET in order to keep information from the form out of the URL and used a Dictionary to store this information.

There is also a try/except block to catch any OS errors when trying to write to this CSV file and write the error(s) in a log file.

The function write_to_csv writes a copy of the information in the form into a CSV file. The implementation can be seen below

From the data provided in the form, I get the name, email, and message and assign it to variables. These variables will be used to send the email and will be written to a CSV file.

The send_email function emails me a copy of all the information contained in the form.

This is done by creating an email object. Then, passing to it all the information that will be in the message.

Using the SMTP library, I start the handshake process with my email provider, pass it my login information (redacted), and finally send the message

Hosting and deploying the site

The project is bound to this Git repository in GitHub.

PythonAnywhere (PA) provides a Terminal in which you are able to set up a virtual environment, install all dependencies, as well as the ability to clone and manage a Git repo. To update my site, all I need to do is commit and push my local changes.

Then, on the PythonAnywhere console, all I need to do is pull the changes and reload the site.

There is a setup process to get this all up and running in PythonAnywhere once all files are ready, but the process is simple and properly explained in their Documentation.

The last step, which is still ongoing, is adding more content to the site. I am incredibly familiar with HTML5 and CSS, so it has been easy to modify the template, add content, add CSS rules, etc.
